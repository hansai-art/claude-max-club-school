# 解釋太技術性，用戶說「我看不懂」

## 問題

Claude 在解釋某個概念或問題時，預設用技術語言——術語、架構圖、縮寫、英文專有名詞堆疊。用戶說「我不懂你在說什麼」，Claude 再解釋一次，還是太技術。

反覆幾輪後用戶放棄理解，只說「你去做就好」。這讓用戶失去了參與決策的機會。

## 原因

Claude 傾向展示「我知道這個技術」，用技術語言證明自己的能力。但用戶要的不是展示，是理解。

另一個原因：Claude 不知道用戶的技術背景，預設了「中等開發者」的基準，但不同用戶差異極大。

## 解決方案

預設用白話文：
- 第一層解釋：用日常比喻，國中生能懂
- 第二層：用戶問「為什麼」才加技術細節
- 複雜概念：表格 + 文字圖解，不是長段落

判斷標準：「這段話，完全不懂技術的人能看懂嗎？」→ 不能 → 重寫。

## 可複用的 CLAUDE.md 規則

```markdown
## 解釋的預設語言層級

- 預設：白話文，用日常比喻，國中生能懂
- 技術細節：用戶主動問「為什麼」或「怎麼運作」才給
- 複雜概念：表格 + 文字圖解，不用長段落

自我檢查：「完全不懂技術的人看得懂這段嗎？」→ 不能 → 重寫
禁止：技術術語堆疊當作預設解釋方式
```
