# 每個階段完成後主動告知下一步

## 問題

Claude 完成一個任務後，靜靜等在那裡，什麼也不說。用戶不知道接下來應該做什麼，或 Claude 到底完成到哪一步了。

特別是多階段的任務（例如：開發 → 測試 → 部署），每個階段完成後如果沒有主動說明，用戶只能靠猜。

## 原因

Claude 把「完成交付物」視為任務終點，沒有把「告知用戶當前狀態和下一步」納入完成的定義。這是一種被動的行為模式——等用戶問才說。

## 解決方案

每個階段完成後，必須主動輸出狀態報告，格式如下：

> 「下一步是 **X**，需要你做的決定是 **Y**，我的判斷是 **Z**，要繼續嗎？」

如果不需要用戶決定，就自動進行下一步，但仍要在進行前告知：
> 「這個階段完成。下一步是 X，我直接繼續。」

## 可複用的 CLAUDE.md 規則

```markdown
## 進度回報
- 每個階段完成後 MUST 主動告知：下一步是什麼、需要用戶做什麼決定、我的建議是什麼
- 格式：「下一步是 X，需要你 Y，我的判斷是 Z，要繼續嗎？」
- 不需要用戶決定的步驟：自動進行，但進行前先說明
- 禁止做完就停在那裡等用戶問
```
